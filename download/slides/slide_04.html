PART 3 · AI APPROACH

A calibrated LLM evaluator with structured output and a human in the loop.

One model does the whole pipeline. The hard part isn't picking a model — it's writing the rubric and forcing the model to be honest.

memory

Model & tools

  • GPT-4-class LLM via z-ai-web-dev-sdk — one structured call per batch
  • Next.js 16 API route server-side — SDK never touches the browser
  • JSON schema output — no free-text parsing, ever
  • No embeddings or classifiers for v0.1 — the LLM reasons about BOTH "does this candidate have AWS ECS?" AND "is 4 jobs in 4 years a red flag?" in a single pass. Stitching 4 specialized models would be over-engineering for a prototype.

verified

Quality & reliability

  • Explicit rubric in the prompt — the model isn't guessing what "good" means
  • Structured JSON schema — parsing failures are caught, not silent
  • Calibrated thresholds — Strong Fit (≥80) is intentionally rare. Most candidates land in Possible (60–79) or Pass (<60)
  • Bias guardrails in system prompt — judge on skills, scope, trajectory ONLY; never on name, gender, age, or university prestige
  • Human-in-the-loop — ScreenWise recommends, the recruiter decides. No auto-reject.
  • Future: dual-pass scoring on borderline calls; embeddings for semantic skill matching at scale (so "TS" matches "TypeScript")
