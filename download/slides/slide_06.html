PART 5 · REFLECTION

What I learned shipping this in 5 hours.

build AI TOOLS I USED
The stack behind the stack

Cursor + Claude for scaffolding the Next.js app and shadcn/ui components. GPT-4-class model (via z-ai-web-dev-sdk) as the actual fitment engine inside the prototype. I did NOT use AI to write the rubric or the bias guardrails — those came from reading about real recruiter workflows first.

bolt HOW AI HELPED
10x on the boring parts

API routes, type definitions, component scaffolding — all 10x faster. It also caught edge cases I would have missed (empty resumes, oversized batches). That freed my time for the parts that actually matter: rubric design and prompt calibration.

warning_amber BIGGEST CHALLENGE
Calibration, not capability

The first prompt gave almost everyone 75+. The model wasn't dumb — it was being too polite. Fix: explicit rubric weights, a hard rule that Strong Fit (≥80) should be rare, and a calibration line in the system prompt. V2 correctly flagged only 1 of 5 sample candidates as Strong Fit.

rocket_launch IF I HAD ANOTHER WEEK
Where I'd take it next
  • Bias-audit dashboard (track score distributions across name/gender proxies)
  • Real PDF resume parsing, not just paste
  • Embeddings-based semantic skill matching ("TS" → "TypeScript")
  • Greenhouse / Lever integration + a second-opinion pass on borderline calls
